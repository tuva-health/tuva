Design principles for the live app

  • One dominant job per region Left rail chooses a file. Header explains it. Controls refine it. Table proves it.
  • Status is compact, not loud Indexed, preview, historical, and reference-data states should read as chips and inline banners.
  • Search feels like the core action The query field becomes the visual anchor once a dataset is selected.

Reference data and historical states

  • Reference data Reuse the same shell, but swap the result area for preview-only or download-first patterns when indexing is unavailable.
  • Historical versions Replace the current heavy warning block with a slim amber notice and a shortcut back to the latest indexed version.
  • Cross-tab matches Present them as quiet related cards instead of another large section competing with the table.
